In a time when the latency between data ingest and data analysis is and must continue to dramatically shrink, data teams are looking for ways to extract any level of time savings from their data pipelines. At the same time, the diversity in data types and schemas exponentially grows, leading to an inherent mismatch in how organizations can achieve both fast and accurate analytics and ML modeling. 

Dataframes in Spark help solve for this, as they are inherently based on having a schema. And while you can read semi-structured data like JSON and then pay the “tax” of scanning the data set twice or attempt to parse out explicit schemas, this is not always the most efficient way. There are a few hybrid approaches that enable teams to handle both data complexity and data pipeline latency concerns.
In this talk, Sheel Choksi will explore options for profiling JSON data to understand what is in the data and help inform explicit schemas with the benefits of drift detection around ever-changing schemas.
