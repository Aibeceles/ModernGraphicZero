# GraphMLDifferences

Machine Learning notebooks for graph-based polynomial analysis using Neo4j Graph Data Science (GDS) library and Apache Zeppelin.

## Overview

This project applies Graph Machine Learning techniques to analyze **difference polynomials** represented as a graph structure. The graph is generated by the `ZeroAndDifferences.jar` algorithm, which computes successive differences of polynomials to determine their rational roots.

📺 **Algorithm Explanation**: [YouTube Video - Zeros and Differences](https://www.youtube.com/watch?v=H4dBkofVA4A)

## Graph Structure

Each node in the graph represents a **differenced polynomial** with the following properties:

| Property | Description |
|----------|-------------|
| `determined` | Binary indicator (0/1) - whether the polynomial is fully determined |
| `zero`, `one`, `two`, `three` | Coefficient indicators for polynomial terms |
| `wNum` | Weight number from the creator node |
| `totalZero` | Total count of zero roots |
| `d` | Depth/degree of the polynomial |

The graph depth level indicates the number of rational roots necessary for the polynomial to be determined:
- **Root level (depth 0)**: Constant polynomial - no expected rational roots
- **Level 1**: Linear polynomial - expects 1 rational root
- **Level 2**: Quadratic polynomial - expects 2 rational roots
- And so on...

## Notebooks

### 1. GraphicLablePrediction.json - Node Classification

**Purpose**: Predicts whether a polynomial node is "determined" (has all expected rational roots) or "underdetermined".

#### Workflow:
1. **Graph Creation**: Load the graph into GDS memory with node properties
2. **GraphSage Training**: Generate node embeddings using unsupervised learning
   - Aggregator: Pool
   - Activation: ReLU
   - 75 epochs
3. **Node Classification**: Train a logistic regression model
   - Target: `determined` property
   - Features: `zero`, `one`, `two`, `three`, `wNum`, `totalZero`
   - Validation: 185-fold cross-validation
   - Metric: F1 Weighted Score
4. **Prediction**: Classify nodes and write predictions back to the graph

#### Results:
- **Training Score**: ~82.9% F1 Weighted
- **Test Score**: ~79.2% F1 Weighted

#### Visualization Legend:
- 🔵 **Blue nodes**: Correctly predicted as determined
- 🟢 **Green nodes**: Correctly predicted as underdetermined
- 🔴 **Red nodes**: Incorrectly predicted

---

### 2. GraphicLinkPrediction.json - Link Prediction

**Purpose**: Predicts potential relationships (zMap edges) between polynomial nodes based on their embeddings.

#### Workflow:
1. **Data Preparation**: Convert integer properties to float for GDS compatibility
2. **Graph Creation**: Project nodes with `dd` (degree) property
3. **Train/Test Split**: 
   - 20% holdout for test set
   - Additional 20% split for training validation
   - Negative sampling ratio: 1.0
4. **GraphSage Training**: Generate 64-dimensional embeddings
   - Aggregator: Pool
   - Activation: ReLU
   - Sample sizes: [25, 10, 10, 10]
   - Search depth: 6
5. **Link Prediction Model**: Train with multiple hyperparameter configurations
   - Feature combiners: HADAMARD, L2, COSINE
   - Penalty values: 0.0001, 1.0, 10000.0
   - Metric: AUCPR (Area Under Precision-Recall Curve)
6. **Prediction**: Generate top-N link predictions with probability scores

#### Best Model Configuration:
- Link Feature Combiner: COSINE
- Penalty: 10000.0
- Test AUCPR: ~55.5%

## Requirements

### Software
- **Neo4j** 4.x+ with Graph Data Science (GDS) Library 1.6+
- **Apache Zeppelin** with Neo4j interpreter configured
- **Java** (for running `ZeroAndDifferences.jar`)

### Neo4j GDS Procedures Used
```cypher
-- Graph Management
gds.graph.create()
gds.graph.drop()

-- Embeddings
gds.beta.graphSage.train()
gds.beta.graphSage.mutate()
gds.beta.graphSage.stream()

-- Node Classification
gds.alpha.ml.nodeClassification.train()
gds.alpha.ml.nodeClassification.predict.write()

-- Link Prediction
gds.alpha.ml.splitRelationships.mutate()
gds.alpha.ml.linkPrediction.train()
gds.alpha.ml.linkPrediction.predict.mutate()
gds.alpha.ml.linkPrediction.predict.stream()
gds.graph.writeRelationship()

-- Model Management
gds.beta.model.drop()
```

## Graph Schema

```
(:Dnode)-[:zMap]->(:Dnode)
(:Dnode)-[:CreatedBye]->(:CreatedBy)
```

- **Dnode**: Polynomial difference nodes
- **zMap**: Relationships mapping zero values between polynomials
- **CreatedBy**: Metadata nodes tracking polynomial origins

## Model Performance Notes

### Node Classification
The classification model achieves good accuracy using polynomial properties as features. GraphSage embeddings were explored but not yet fully integrated as feature properties for classification.

### Link Prediction
Link prediction shows moderate performance. The model predicts connections between polynomial nodes based on structural similarity captured by GraphSage embeddings. The COSINE feature combiner with high regularization (penalty=10000) yielded the best results.
